# Настройка GitHub Pages

## ⚠️ Важно: Ограничение для приватных репозиториев

GitHub Pages для **приватных репозиториев** доступен только с **GitHub Enterprise** (платный план).

Для бесплатного использования GitHub Pages репозиторий должен быть **публичным**.

## Варианты решения:

### Вариант 1: Сделать репозиторий публичным (рекомендуется)

1. Перейдите в настройки репозитория:
   - https://github.com/batalovmv/alla/settings

2. Прокрутите вниз до раздела **Danger Zone**

3. Нажмите **Change visibility** → **Change to public**

4. Подтвердите изменение

5. После этого вернитесь в **Settings → Pages** и настройте:
   - Source: **Deploy from a branch**
   - Branch: **gh-pages**
   - Folder: **/ (root)**

6. Сайт будет доступен по адресу:
   **https://batalovmv.github.io/alla/**

### Вариант 2: Использовать GitHub Enterprise (платно)

Если нужен приватный репозиторий, можно:
- Обновить аккаунт до GitHub Enterprise
- Или использовать бесплатный пробный период (30 дней)

### Вариант 3: Альтернативный хостинг (бесплатно для приватных репозиториев)

Можно использовать другие платформы, которые поддерживают приватные репозитории:

#### Netlify (рекомендуется)
1. Зарегистрируйтесь на https://netlify.com
2. Подключите GitHub репозиторий
3. Настройки:
   - Build command: `npm run build`
   - Publish directory: `dist`
4. Сайт будет доступен по адресу типа: `your-site.netlify.app`

#### Vercel
1. Зарегистрируйтесь на https://vercel.com
2. Подключите GitHub репозиторий
3. Автоматически определит настройки Vite
4. Сайт будет доступен по адресу типа: `your-site.vercel.app`

#### Cloudflare Pages
1. Зарегистрируйтесь на https://pages.cloudflare.com
2. Подключите GitHub репозиторий
3. Настройки:
   - Build command: `npm run build`
   - Build output directory: `dist`

## Рекомендация

Для личного проекта косметолога **рекомендуется сделать репозиторий публичным** - это бесплатно и не требует дополнительных настроек. Код сайта обычно не содержит конфиденциальной информации.

Если в коде есть чувствительные данные (API ключи, пароли), их нужно вынести в переменные окружения и добавить в `.gitignore`.
